### Step 4 — Sole-root simple command requirement

The parsed tree MUST be exactly one node, kind == "command", with NO surrounding list / pipeline / subshell / compound / background. Anything else → BLOCK.

```python
if len(trees) != 1:
    BLOCK("must be a single command, not a compound/list/pipeline")
root = trees[0]
if getattr(root, "kind", None) != "command":
    BLOCK(f"must be a simple command, got {root.kind}")
# Note: bashlex represents `cmd &` and `cmd | cmd` as list/pipeline parents
# whose children are commands; root.kind != "command" catches both.
```

### Step 5 — argv strict white-list

```python
APPROVED_ARGS = {"-m", "--message", "-F", "--file",
                 "--amend", "--no-edit", "--no-gpg-sign",
                 "--allow-empty-message"}

# 5a. No env assignments (use node.assignments, NOT a filter on parts)
if getattr(root, "assignments", None):
    BLOCK("env-prefixed commit not allowed")

# 5b. No command/process substitution anywhere in argv words
for p in getattr(root, "parts", []):
    for sub in getattr(p, "parts", []) or []:
        if getattr(sub, "kind", None) in ("commandsubstitution", "processsubstitution"):
            BLOCK("command/process substitution in argv not allowed")

# 5c. argv strict shape
argv = [getattr(p, "word", "") for p in getattr(root, "parts", [])
        if getattr(p, "kind", None) == "word"]
if len(argv) < 2 or argv[0] != "git" or argv[1] != "commit":
    # Special case: not a commit invocation (e.g. `git status`) → PASS
    # We only inspect commits; non-commit gits exit early here.
    if argv and argv[0] == "git" and len(argv) >= 2 and argv[1] != "commit":
        # Walk the rest looking for embedded `commit` (e.g. `git -c alias.ci=commit ci`)
        # Any subcommand other than literal `commit` with `git -c` involved → BLOCK
        if "-c" in argv[1:]:
            BLOCK("`git -c ...` form not allowed (alias injection risk)")
        exit 0  # PASS — non-commit git command
    if argv[0] != "git":
        exit 0  # PASS — not a git command at all
    BLOCK("malformed git commit invocation")

# 5d. Each argv[2:] token must be in APPROVED_ARGS (or be the operand of -m/-F/--message/--file)
i = 2
has_message_or_amend = False
while i < len(argv):
    tok = argv[i]
    if tok in ("-m", "-F", "--message", "--file"):
        if i + 1 >= len(argv):
            BLOCK(f"option {tok} missing argument")
        i += 2
        has_message_or_amend = True
        continue
    if tok.startswith("--message=") or tok.startswith("--file="):
        i += 1
        has_message_or_amend = True
        continue
    if tok == "--amend":
        i += 1
        has_message_or_amend = True
        continue
    if tok in APPROVED_ARGS:
        i += 1
        continue
    BLOCK(f"option {tok!r} not in approved white-list")

if not has_message_or_amend:
    BLOCK("commit must include -m/-F/--message/--file or --amend")
```

### Step 6 — Marker validation

Marker file: `~/.claude/hooks/.review-passed.json` (single-use).

Schema v=1:
```json
{
  "schema_version": 1,
  "repo_id":  "<absolute path of git common dir, Path.resolve()>",
  "head_oid": "<output of git rev-parse HEAD at review time>",
  "tree_sha": "<output of git write-tree at review time>",
  "ts": <unix timestamp>
}
```

Hook validation (Python, no shelling through xargs/realpath):
```python
import json, os, time, subprocess
from pathlib import Path

MARKER = Path.home() / ".claude/hooks/.review-passed.json"

# 1. Read JSON
try:
    payload = json.loads(MARKER.read_text())
except FileNotFoundError:
    BLOCK("no review marker")
except (OSError, json.JSONDecodeError):
    BLOCK("marker unreadable / not JSON")

# 2. Schema version
if payload.get("schema_version") != 1:
    BLOCK("unrecognized marker schema_version")

# 3. TTL (use mtime AND payload.ts; reject if either is stale)
mtime = MARKER.stat().st_mtime
if time.time() - mtime > 1800 or time.time() - payload["ts"] > 1800:
    BLOCK("marker expired")

# 4. repo_id (Path.resolve, not xargs realpath)
def run(cmd, timeout=5):
    return subprocess.run(cmd, capture_output=True, text=True, timeout=timeout, check=True).stdout.strip()

try:
    cur_common_dir = run(["git", "rev-parse", "--git-common-dir"])
    cur_repo_id = str(Path(cur_common_dir).resolve())
except (subprocess.SubprocessError, subprocess.TimeoutExpired):
    BLOCK("cannot determine current repo identity")
if payload["repo_id"] != cur_repo_id:
    BLOCK("marker is for a different repo")

# 5. head_oid
try:
    cur_head = run(["git", "rev-parse", "HEAD"])
except (subprocess.SubprocessError, subprocess.TimeoutExpired):
    BLOCK("cannot determine current HEAD")
if payload["head_oid"] != cur_head:
    BLOCK("HEAD has moved since review (branch switch, rebase, etc.)")

# 6. tree_sha
try:
    cur_tree = run(["git", "write-tree"])
except (subprocess.SubprocessError, subprocess.TimeoutExpired):
    BLOCK("cannot validate index state")
if payload["tree_sha"] != cur_tree:
    BLOCK("staged content has changed since review")

# 7. Single-use: atomic unlink, then PASS
MARKER.unlink()
exit 0  # PASS
```

### Step 7 — K_CLASS_SENTINEL_PROHIBITION text

```
**K-class red line — code review marker integrity**

Subagents and main session must NEVER:
1. Wrap, hide, or indirect-invoke `git commit` in any way that defeats the
   hook (compound `&&`/`;`/`||`, subshells `(…)`, background `&`, pipelines `|`,
   command/process substitutions `$(…)`/`<(…)`, `eval`, `bash -c`, `python -c`,
   `command/env/nice/...` wrappers, command aliases like `git ci`).
2. Run a non-plain `git commit` form: `-a/--all`, `--include`, `--only`,
   `-p/--patch`, pathspec, `-c <key>=<val>`, `-C <path>`, `--git-dir=...`,
   `--work-tree=...`, prefix env assignments (`PATH=. git commit`, etc.).
3. Manipulate marker / hook / vendor files: write, append, copy, chmod, touch,
   rename, atomic-replace `~/.claude/hooks/.review-passed.json`,
   `~/.claude/hooks/_vendor/*`, `~/.claude/hooks/pre-commit-review.{sh,py}`.
4. Use `--no-verify`, `core.hooksPath` override, `-c alias.*=`, or any other
   git-config-based hook bypass.

Approved flow: reviewer agent runs → reviewer agent writes marker via the helper
→ main session runs PLAIN `git commit -m "..."` (or `-F`, or `--amend`) in the
target repo's CWD → hook validates and consumes marker.

Exception: when the user explicitly authorizes hook/vendor maintenance work
(e.g. v0.8.3 P0.0), the K-class file-modification prohibition is lifted for
that scope only, with user attestation in PRD.
```
