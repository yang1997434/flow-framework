**Conclusions**:
- bashlex DOES dequote static quoting (`""`, `\g`, `gi""t`) → your Round 2 B.1' claim "bashlex words are not safely dequoted" is FALSE for static quoting.
- bashlex DOES NOT expand runtime constructs (`$VAR`, `$(...)`, `${...}`, brace expansion, ANSI `$'...'`) — argv contains the literal pre-expansion form.
- bashlex separates leading env assignments into `node.assignments` so argv is clean.
- Command substitution `$(...)` stays embedded in the word, not exposed as argv entries — walker must recurse word.parts to find substitution AST nodes.

## Revised white-list design (D'''+W)

### Step 1: Pre-screen

Cheap regex: `\bgit\b` — if no `git` word in command, exit 0 PASS.

(Removed `commit` requirement: walker now decides subcommand, not pre-screen.)

Cost: any command containing `git` (e.g. `git status`, `git push`) pays bashlex import (~100ms). Acceptable per F.3 ("correctness should win").

### Step 2: Pre-screen for command size

If command bytes > 1MB → BLOCK with reason "command too large to analyze; simplify" (F.2 fix).

### Step 3: bashlex parse

Try `bashlex.parse(command)`. On any exception:
- If pre-screen said "git word present" → BLOCK with reason "Cannot safely analyze; please simplify (e.g. avoid quoted heredocs containing git words; pre-stage and run plain `git commit` separately)".

### Step 4: Recursive AST walk for any commit invocation

```python
def find_commit_invocation(node) -> tuple[str, ...] | None:
    """Return tuple of argv if any node in tree is `git commit ...`, else None.
    Recursive — handles list/pipeline/subshell/background/word substitutions."""

    if isinstance(node, list):
        for n in node:
            r = find_commit_invocation(n)
            if r: return r
        return None

    kind = getattr(node, "kind", None)
    if kind == "command":
        # collect argv from word parts
        argv = [getattr(p, "word", "") for p in getattr(node, "parts", [])
                if getattr(p, "kind", None) == "word"]
        # check assignments — non-empty means env-prefixed: BLOCK regardless of subcommand
        assigns = [getattr(p, "word", "") for p in getattr(node, "parts", [])
                   if getattr(p, "kind", None) == "assignment"]
        # also recurse into word.parts for embedded command substitutions
        for p in getattr(node, "parts", []):
            for sub in getattr(p, "parts", []) or []:
                r = find_commit_invocation(sub)
                if r: return r
        # is this command a git invocation?
        if argv and is_git_command_name(argv[0]):
            # is it `git commit`? (literal subcommand match, no alias resolution)
            if len(argv) >= 2 and argv[1] == "commit":
                return ("__plain__", *argv, *(f"ENV={a}" for a in assigns))
            # `git -c X commit` / `git -C X commit` etc. — argv[1]=='-c' / '-C': also treat as commit
            # (we BLOCK these regardless — captured below)
            for i, w in enumerate(argv[1:], start=1):
                if w == "commit":
                    return ("__nonplain__", *argv, *(f"ENV={a}" for a in assigns))
        return None

    # composite kinds — recurse
    for child_attr in ("parts", "list", "commands"):
        children = getattr(node, child_attr, None) or []
        for c in children:
            r = find_commit_invocation(c)
            if r: return r
    return None


def is_git_command_name(w: str) -> bool:
    """Match `git` as a literal command name, allowing common path prefixes."""
    if w == "git":
        return True
    if w.endswith("/git") and not any(c in w for c in "$`(){}[]?*"):
        return True
    return False
```

### Step 5: White-list validation on commit invocation argv

If find_commit_invocation returned a tuple:

```python
APPROVED_OPTIONS = {
    "-m", "--message",
    "-F", "--file",
    "--amend",
    "--no-edit",
    "--no-gpg-sign",
    "--allow-empty-message",
}

def is_white_listed(token, argv) -> tuple[bool, str | None]:
    """Determine if the captured commit invocation is in the white list."""
    flavor, *rest = token  # flavor is "__plain__" or "__nonplain__"

    # Reject env-prefixed commits (assignments captured as ENV=...)
    if any(a.startswith("ENV=") for a in rest):
        return False, "env-prefixed commit not allowed (use plain git commit after `unset` if needed)"

    # Reject any non-plain shape (--git-dir, -c, -C, /path/git, git$VAR, etc.)
    if flavor == "__nonplain__":
        return False, "git global options or non-`git` argv[0] not allowed; use plain `git commit` in target repo"

    # `rest` is `[git, commit, ...]` for __plain__ — strip and inspect args
    args = list(rest[2:])
    # `--amend` may appear with no other args; that's fine
    if args == ["--amend"] or args == ["--amend", "--no-edit"]:
        return True, None
    # otherwise must use one of: -m TEXT / --message=TEXT / --message TEXT / -F PATH / --file=PATH / --file PATH
    has_message = False
    i = 0
    while i < len(args):
        a = args[i]
        if a in ("-m", "-F", "--message", "--file"):
            if i + 1 >= len(args):
                return False, f"option {a} missing argument"
            i += 2
            has_message = True
            continue
        if a.startswith("--message=") or a.startswith("--file="):
            i += 1
            has_message = True
            continue
        if a in APPROVED_OPTIONS:
            i += 1
            continue
        return False, f"option {a!r} not in approved white-list"

    if not has_message and "--amend" not in args:
        return False, "commit must include -m/--message/-F/--file or be --amend"

    return True, None
```

### Step 6: Marker validation (when white-list passes)

Marker file: `~/.claude/hooks/.review-passed.json` (single-use).

```json
{
  "schema_version": 1,
  "repo_id": "<absolute path of git common dir>",
  "tree_sha": "<git write-tree output at review time>",
  "ts": <unix timestamp>
}
```

Hook validation:
1. Read JSON. BLOCK on parse / missing / `schema_version != 1`.
2. BLOCK if `time.time() - mtime > 1800` seconds.
3. BLOCK if `marker.repo_id != $(git rev-parse --git-common-dir | xargs realpath)`. (Note: hook computes against the CWD it runs in, which is the same CWD as the git commit invocation, since assignments and -C/-c are blocked at white-list step.)
4. BLOCK if `marker.tree_sha != $(timeout 5 git write-tree)`. On write-tree error/timeout → BLOCK with reason "cannot validate index".
5. On all checks pass → atomically `os.unlink(marker)` (single-use) → exit 0 PASS.
