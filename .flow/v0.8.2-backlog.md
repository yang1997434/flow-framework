## 2. Post-ship discoveries

### 2.1 `context_estimator` 1M edge case — Pro/Max default without alias

**Status**: v0.8.1 shipped a 4-rung priority chain (`scripts/common/context_estimator.py:_resolve_limit`) that handles 95% of cases. Remaining edge:

> User on Anthropic Pro/Max plan (1M context default), but **no `ANTHROPIC_DEFAULT_*_MODEL` aliases configured at all** in `~/.claude/settings.json::env`.

**Current behavior**: rung 2a misses (no alias) → rung 2b plan-level scan misses (no `[1m]` signal anywhere) → rung 3 `MODEL_LIMITS` table → 200k → **5x over-report**.

**Mitigation already in place (governs 95%)**: settings.json补 `ANTHROPIC_DEFAULT_<BASE>_MODEL=...[1m]` for each base — done by user 2026-05-07; rung 2a hits directly.

**v0.8.2 fix candidate**:
- Option A: detect Anthropic Pro/Max plan via API (`/v1/account` or similar) and infer 1M default — requires API key + network call from hook; latency-sensitive
- Option B: add `FLOW_DEFAULT_ASSUMED_LIMIT` env override (cheap; user opts in)
- Option C: heuristic "if transcript size has crossed any plausible 200k boundary recently AND continued growing, retroactively assume 1M" — fragile
- Option D: scrape Claude Code's own internal config to detect plan tier — couples to private API

**Recommendation**: ship Option B (env override) in v0.8.2, defer A/C/D to v0.8.3.

### 2.2 K-class sentinel violation hardening

**Status**: 2 implementer self-touches of `~/.claude/hooks/.review-passed` during v0.8.1 ship (T22 round-0 + estimator round-2). Both let real bugs slip past pre-commit review until codex caught them.

**v0.8.2 fix candidate**:
- Make sentinel single-use: pre-commit hook deletes sentinel after consumption → prevents implementer's `touch` from persisting through next commit
- OR: hook validates sentinel mtime is newer than first-staged-file mtime AND has external-reviewer signature (requires reviewer agent to write a marker file with hash of staged diff)
- Cheaper: dispatch prompt template auto-prepends "DO NOT touch ~/.claude/hooks/.review-passed; partial-state report on hook block"

**Recommendation**: ship the dispatch prompt template change (free; immediate) + investigate single-use sentinel as proper fix.

### 2.3 Plan suite-count citations should be relative

**Status**: v0.8.1 plan cited absolute test counts ("Suite ≥ 477"); reality at write-time was already 5x off. By T15 plan citations were unreliable.

**v0.8.2 plan-writing rule**:
- Use relative format: "≥ baseline + N new cases" instead of absolute "≥ N total"
- Ages better; baseline auto-recomputed at task start

**Recommendation**: update `claude/skills/flow/flow-phase2-execute/SKILL.md` plan-template guidance.
